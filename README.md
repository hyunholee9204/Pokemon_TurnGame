# 🎮 Web-Based Pokémon Strategy Turn-Based Battle Game

> 개발 기간: 2026/04/25~2026/06/09 1인 개발

> **20종의 시너지 유물 시스템과 진형별 HP 분배 공식 알고리즘을 적용한 전략 턴제 배틀 시뮬레이터**입니다.  
> 단순한 전투 구현을 넘어, 로그라이크 요소와 기획적 밸런싱 최적화에 초점을 맞추어 개발되었습니다.

---

## 🚀 Key Features (핵심 기능)

- **정교한 턴제 배틀 메커니즘:** 포켓몬 스피드(Speed) 스탯에 따른 유기적인 턴 액션 순서 계산 및 배틀 연산 루틴 구현
- **20종의 고유 유물(Relic) 시스템:** 보호막, 출혈, 선공권, 부활, 골드 보너스 등 다양한 시너지를 창출하는 로그라이크 아이템 메커니즘 설계
- **노드 기반 스테이지 진행:** 진행도(`left` 값) 및 노드 타입별 난이도 자동 스케일링 엔진 구축

---

## 💡 System Design & Balancing (기획 및 밸런싱 의도)

### 1. 진형 배치에 따른 스탯 반비례 공식 (`1 > 4 > 2 = 3`)
유저에게 전략적인 타겟팅 턴 소모를 유도하고 전투의 긴장감을 유지하기 위해 진형별 스탯을 정교하게 분배했습니다.
- **1번 (선두 탱커):** 높은 HP/보호막 배치로 아군 파티의 전열을 방어 (스피드 최하)
- **2, 3번 (중앙 딜러):** '유리 대포(Glass Cannon)' 컨셉으로 최하급 HP를 부여하되, 최고급 스피드를 부여하여 적에게 선공 위협을 가함
- **4번 (후미 서포터):** 전투 지속력을 위해 중상급 HP와 특수 유틸리티 스킬 매칭

### 2. 스테이지 동적 난이도 및 밸런싱 풀(Pool)
동일한 진행도(`left` 값)와 종류를 가진 일반 전투 노드(`NORMAL`) 간의 불합리한 난이도 격차를 줄이기 위해, **적군 파티 전체의 총 HP 합산 풀(Pool)을 통일**시킨 후 포켓몬 조합에 따라 스탯을 유기적으로 분배하는 균형 알고리즘을 적용했습니다.

### 💻 Development Languages & Environment
- **Frontend:** HTML5, CSS3, JavaScript (ES6+) — 웹 기반 배틀 UI 인터페이스 및 동적 렌더링 구현
- **Backend/Logic:** Java — 전투 핵심 연산 및 시스템 비즈니스 로직 설계
- **Data Format:** JSON — 포켓몬 스탯 프리셋, 스킬 프리셋, 유물(Relic) 효과 스펙 데이터 구조화 및 관리

### 📊 Database: `pokemon`

| 테이블명 | 역할 및 설명 | 주요 관리 데이터 |
| :--- | :--- | :--- |
| **`users`** | 사용자 계정 메인 엔티티 | 유저 고유 ID, 계정 정보, 생성일 등 |
| **`game_slots`** | 유저별 세이브 파일 관리 슬롯 | 진행도(Stage/left), 보유 재화(G), 슬롯 활성화 상태 |
| **`pokedex`** | 유저가 인게임 전투에 출전시킬 4마리의 포켓몬 진형 | 인스턴스 포켓몬 ID, 배치 순서(1~4번), 현재 상태 데이터 |
| **`pokemon_dict`** | 마스터 데이터: 전체 포켓몬 도감 | 포켓몬 이름, 베이스 스탯(HP, Speed 등), 진화 단계 |
| **`skill_pedia`** | 마스터 데이터: 포켓몬 전용 스킬 사전 | 스킬 고유 ID, 데미지 계수, 소모 코스트, 디버프(출혈 등) 속성 |
| **`relic_pedia`** | 마스터 데이터: 유물 상점 및 인게임 유물 리스트 | 유물 고유 ID, 유물 명칭, 특수 효과 메커니즘, 구매 가격 |
